如何生成中高保真 HTML 原型

把需求、截图和参考页面转化为可预览、可迭代的 HTML 原型。

适用场景 场景判断

当用户提供需求文档、页面截图、参考作品或口头描述，希望生成中高保真 HTML 原型时使用。

输入材料

  • 需求文档或需求描述
  • 当前页面截图
  • 参考页面或风格要求
  • 用户对布局、交互、信息密度的偏好

执行流程

  1. 阅读需求，提取页面目标
  2. 拆分页面模块和信息层级
  3. 确定布局结构和交互入口
  4. 生成 HTML 原型
  5. 检查视觉质量和概念清晰度
  6. 根据反馈迭代并沉淀版本

AI 分工与人工确认

AI 负责提炼需求、生成结构、输出 HTML 和总结报告。
用户负责判断页面是否好看、功能是否真实、信息是否过密。

判断标准

  • 页面接近中高保真，不是粗糙线框
  • 信息层级清楚，不堆功能
  • 交互入口明确，能继续迭代开发
  • 输出结果符合任务上下文和模板规则

复用提示词

请基于当前任务上下文、需求文档和参考截图，按照“页面目标 → 模块拆分 → 布局结构 → 交互说明 → HTML 原型 → 检查清单”的流程，生成中高保真 HTML 原型。不要输出粗糙线框图。
