Skill 化入口

Skill 化目的

把当前任务中的方法、步骤、判断标准和 AI 协作规则提取出来，形成下次可复用的 Skill。

提取来源

  • 任务上下文
  • HTML 成果
  • 当前任务报告
  • 流程图节点
  • 当前整理内容

生成后动作

可继续编辑 Skill 草稿，并保存到 Skill 库，供后续“从 Skill 创建任务”复用。

Skill 草稿：如何设计任务方法沉淀平台的任务区原型

适用场景

当用户需要为“任务方法沉淀平台”设计任务区页面，并希望在一个页面中承载任务执行、报告生成和方法沉淀能力时使用。

输入材料

  • 任务需求文档
  • 当前页面截图或旧版原型
  • 产品定位与边界
  • 已有首页原型或设计风格

执行流程

  • 阅读需求，明确任务区定位
  • 拆解任务区核心视图
  • 规划三栏工作视图结构
  • 定义任务报告页面
  • 定义流程图页面
  • 定义 Skill 化页面
  • 输出 HTML 原型并迭代优化

AI 分工

AI 负责梳理结构、生成 HTML 原型、总结任务报告和提取 Skill 草稿；用户负责判断信息层级、审美表现和产品概念是否清晰。

判断标准

  • 页面结构清晰，不堆功能
  • 每个视图边界明确
  • 当前任务报告可模板化生成
  • 流程图和 Skill 化具有后续扩展空间

输出物

任务区 HTML 原型、当前任务报告、流程图结构、Skill 草稿。
