## 2. 产品定位

### 2.1 一句话定位

本平台是一个面向个人的任务方法沉淀与 AI Skill 生成平台，用于把任务上下文、HTML 成果、任务报告、流程图和复盘经验沉淀为可复用的任务流程。

### 2.2 产品不是什么

本平台不是普通任务管理工具，不以待办事项和完成率为核心。

本平台不是通用知识库，不沉淀快捷键、语法、工具菜单、基础概念等可直接询问 AI 的内容。

本平台不是自由 AI 对话工具，不以聊天记录为核心，也不追求保存所有 AI 对话过程。

### 2.3 产品真正沉淀什么

平台重点沉淀以下内容：

1. 一类任务的完成思路。
2. 一类任务的执行步骤。
3. 完成任务时的判断标准。
4. 人与 AI 的协作规则。
5. 可复用的输出模板。
6. 任务执行过程中的关键复盘。
7. 可转化为 Skill 的方法流程。

### 2.4 核心价值

平台的核心价值不是保存内容，而是把任务完成过程结构化。

用户每完成一次任务，都可以沉淀：

```text
任务上下文
↓
任务执行材料
↓
阶段性成果
↓
当前任务报告
↓
流程图
↓
可复用 Skill
```

最终形成从“做一次任务”到“沉淀一种方法”的闭环。

## 3. 用户与使用场景

### 3.1 目标用户

适合经常使用 AI 完成复杂任务的个人用户，包括但不限于：

1. 产品经理。
2. 售前工程师。
3. 方案人员。
4. 原型设计人员。
5. 内容运营人员。
6. 咨询顾问。
7. 经常用 AI 生成文档、方案、HTML 原型和流程图的人。

### 3.2 典型场景

#### 场景一：生成 HTML 原型

用户有需求文档、截图或参考页面，希望让 AI 生成中高保真 HTML 原型，并把整个任务流程沉淀为以后可复用的方法。

#### 场景二：整理当前任务报告

用户完成某个任务阶段后，需要将任务上下文、HTML 成果、关键结论、行动项和流程图建议整理成结构稳定的报告。

#### 场景三：从任务提炼 Skill

用户完成一次任务后，希望平台自动提取任务完成流程、AI 分工、人工确认点和输出标准，形成可复用 Skill。

#### 场景四：复用已有 Skill 创建任务

用户下次遇到类似任务时，可以从 Skill 库选择已有 Skill，快速创建带有默认上下文、步骤、检查清单和输出模板的新任务。

## 4. 产品一级结构

平台包含四个一级页面：

```text
首页
任务区
Skill 库
设置
```

### 4.1 首页

首页是工作入口，用于查看日历、今日任务、历史任务、最近沉淀和最近使用 Skill。

### 4.2 任务区

任务区是单个任务的核心工作台，用于填写上下文、保存 HTML 成果、生成当前任务报告、生成流程图，并将任务转化为 Skill。

### 4.3 Skill 库

Skill 库用于管理已沉淀的方法流程，每个 Skill 都是一套可复用任务方法。

### 4.4 设置

设置页是规则中枢，用于配置任务报告模板、当前整理模板、Skill 生成模板、本地 AI 输入/回传模板和本地保存路径。

## 8. 工作视图需求

### 8.1 页面定位

工作视图用于收集任务材料、保存 HTML 成果、查看预览和生成当前任务报告。

### 8.2 页面布局

工作视图采用三栏布局：

| 区域 | 内容                             |
| -- | ------------------------------ |
| 左侧 | 任务素材、任务上下文、HTML 粘贴入口、HTML 成果列表 |
| 中间 | 当前成果查看区，包括 HTML 预览和任务报告预览      |
| 右侧 | 当前整理，按固定模板展示                   |

### 8.3 左侧任务素材

#### 8.3.1 报告输入预览

报告输入预览用于展示 AI 生成任务报告时会读取哪些内容。

包括：

1. 报告对象。
2. 模板规则。
3. 参考材料。
4. 生成内容。

该区域不是手填表单，而是根据任务上下文、HTML 成果和当前整理自动生成。

#### 8.3.2 任务上下文

任务上下文由用户填写，包含：

1. 任务目标。
2. 背景资料。
3. 约束条件。
4. 判断标准。

#### 8.3.3 HTML 粘贴入口

用于粘贴外部 AI 或工具生成的 HTML 代码。

要求：

1. 入口轻量，不做重型代码编辑器。
2. 用户主要查看预览，不需要阅读代码。
3. 支持临时预览。
4. 支持保存为 HTML 成果。
5. 支持添加备注。
6. 支持是否纳入任务报告。

#### 8.3.4 HTML 成果列表

每个 HTML 成果包含：

1. 成果名称。
2. HTML 内容。
3. 成果说明。
4. 是否纳入报告。
5. 创建时间。
6. 更新时间。
7. 当前预览状态。

### 8.4 中间成果查看区

包含：

1. HTML 预览。
2. 当前任务报告预览。
3. 报告流程图建议。

### 8.5 右侧当前整理

当前整理必须按照固定模板展示，不是自由笔记。

默认字段：

1. 当前结论。
2. 关键依据。
3. 待确认问题。
4. 行动项。
5. 流程图节点。

AI 可以建议填写内容，但用户确认后才能保存为沉淀内容。

## 9. 当前任务报告需求

### 9.1 页面定位

当前任务报告用于将单个任务的上下文、HTML 成果、执行过程、当前结论、风险边界和后续行动整理成正式报告。

### 9.2 默认报告模板

当前任务报告默认包含六章：

| 章节 | 名称         | 说明                |
| -- | ---------- | ----------------- |
| 一  | 任务概述       | 当前任务要解决什么问题       |
| 二  | 上下文依据      | 任务目标、背景、约束、判断标准   |
| 三  | HTML 成果说明  | 已保存 HTML 原型的用途和价值 |
| 四  | 当前结论       | 本轮明确判断和方案方向       |
| 五  | 风险边界       | 不做什么、不能误解什么       |
| 六  | 后续行动与流程图建议 | 行动项和流程图节点建议       |

### 9.3 报告生成规则

AI 生成报告时必须遵守：

1. 不得新增章节。
2. 不得删除必填章节。
3. 不得修改章节名称。
4. 不得输出无法解析的长文本。
5. 不得编造缺失信息。
6. 信息不足时标注“待补充”。
7. 已被用户确认的字段不可直接覆盖。

### 9.4 报告生成流程

```text
读取任务上下文
↓
读取 HTML 成果
↓
读取当前整理
↓
读取报告模板
↓
AI 生成报告
↓
平台校验结构
↓
回写报告页和当前整理
↓
用户修订
↓
保存报告
```

### 9.5 报告校验

平台需要校验：

1. 六章是否完整。
2. 字段名称是否一致。
3. 必填字段是否为空。
4. 行动项是否可解析。
5. 流程图节点是否可生成。
6. 是否存在模板外字段。

校验失败时提示：

```text
AI 返回结果不符合当前报告模板，请重新生成或手动修正。
```

5. 节点样式配置。
6. 多流程图管理。
7. 从流程图反向更新 Skill 执行流程。

节点字段：

1. id。
2. title。
3. description。

6. position_y。
7. sort_order。

连线字段：

1. id。
2. source_node_id。
3. target_node_id。
4. label。

## 16. 非功能需求

### 16.1 本地优先

所有核心数据必须保存在本地 SQLite 数据库和本地文件系统中。

### 16.2 自动保存

要求：

1. 表单输入自动保存。
2. 失焦时触发保存。
3. 切换页面前触发保存。
4. 服务重启后数据不丢失。

### 16.3 备份机制

要求：

1. 每次启动自动备份。
2. 生成任务报告前自动备份。
3. 保留最近 N 份备份。
4. 支持手动备份。
5. 支持恢复备份。
6. 恢复前自动生成当前快照。

### 16.4 模板稳定性

模板是平台核心能力。

要求：

1. 模板支持版本管理。
2. 每次生成报告记录模板版本。
3. 每次生成 Skill 记录模板版本。
4. AI 输出必须通过结构校验。
5. 模板变更不能破坏旧报告。

### 16.5 安全与数据边界

要求：

1. 默认本地运行。
2. 用户可配置是否连接本地 AI 服务。
3. 不默认上传用户数据。
4. 导出任务包时需明确导出内容范围。

## 19. 关键设计原则

### 19.1 不沉淀基础知识

快捷键、语法、工具菜单、普通概念解释不作为核心沉淀对象。

### 19.2 沉淀任务方法

平台沉淀的是完成任务的流程、判断、步骤、输出标准和复盘经验。

### 19.3 以模板约束 AI

AI 不能自由发挥，必须按任务报告模板、整理模板、Skill 模板和回传 JSON 输出。

### 19.4 用户保留最终判断权

AI 可以生成建议，但当前结论、行动项、流程图节点、Skill 内容必须经过用户确认。

### 19.5 从任务走向 Skill

每个任务不是孤立记录，最终要服务于可复用方法的生成。

## 20. 最终目标

平台最终要实现：

```text
把一次任务经验沉淀为可复用流程，
再把流程转化为 AI 可以执行的 Skill，
让下一次同类任务可以更快、更稳、更可控地完成。
```
